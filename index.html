    Brando:: Brand Governance for AI

    Brando Schema v1.3

    Brando is an open-source vocabulary for structuring brand identity, governance policies, and content guidelines in machine-readable formats. It enables AI systems and Large Language Models to retrieve and apply brand rules consistently across automated workflows.

    Published under Creative Commons Attribution 4.0 (CC BY 4.0) with W3C Royalty-Free Patent Policy protections, Brando extends Schema.org's Brand vocabulary to support structured brand representation across verbal identity, visual identity, audio branding, strategic positioning, governance, and approved assets.

    What Brando Is

    An open vocabulary — not a platform, not software, not a service.

    Brando defines 90+ properties for describing brand elements as structured data:

    • Verbal Identity: toneOfVoice, writingStyle, approvedTerms, prohibitedTerms
    • Visual Identity: colourPalette, typography, logo, imageryStyles
    • Audio Identity: audioLogo, voiceSignature, pronunciationGuide
    • Strategic Positioning: missionStatement, visionStatement, coreValues
    • Governance: guardRails, refusalStrategies, enforcementLevel
    • Assets: Approved logos, templates, fonts, reference materials

    These properties extend Schema.org's Brand type to make brand guidance machine-readable for AI systems.

    How It Works

    1. Structure Brand Data

    Brand strategists use Brando Schema vocabulary to express brand policies as:

    • JSON-LD (canonical format, extends Schema.org)
    • YAML with Markdown (human-readable, Git-friendly)
    • JSON sidecars (for tooling compatibility)

    Example:

    {
  "@context": "https://brandoschema.com/",
  "@type": "Brand",
  "name": "Acme Corp",
  "toneOfVoice": "confident, clear, and approachable",
  "guardRails": ["Never make medical claims", "Always cite sources"]
}

    2. Store as Files

    Policy files live in version-controlled repositories — no database, no platform, just files.

    3. Serve to AI Systems

    Organisations implement retrieval layers (RAG systems, MCP servers, API endpoints) to serve policy files to AI workflows.

    4. AI Systems Retrieve and Apply

    LLMs, agents, and content generation pipelines query the structured brand data and apply policies at runtime.

    No vendor lock-in. The schema is open. The files are portable. Implementation is up to you.

    What Brando Is NOT

    • Not a platform — it's a data vocabulary
    • Not software — it's a schema specification
    • Not a service — it's a standard anyone can implement
    • Not proprietary — it's CC BY 4.0 open source

    Think of it like:

    • Schema.org for product data
    • OpenAPI for API specifications
    • BPMN for process diagrams

    Brando is for brand policies.

    Implementation

    Organisations implement Brando-structured brand data in different ways:

    Common patterns:

    • File-based RAG retrieval (Markdown/YAML policies in Git)
    • MCP servers exposing brand policies to Claude/ChatGPT
    • Custom APIs serving JSON-LD brand data to applications
    • CMS integrations for content governance

    Infrastructure choices:

    • Self-hosted (DigitalOcean, AWS, Azure, on-prem)
    • Managed services (any RAG platform, vector DB, CMS)
    • Hybrid (Git + CDN for static policies)

    Brando is infrastructure-agnostic — use whatever stack works for you.
