    Brando:SyntheticPersona

    Brand Oracle (Brando) Schema Vocabulary v1.4 (Draft)

    Parent: schema:Intangible
    Namespace: https://brandoschema.com/

    Definition:
    A governed, brand-controlled digital persona (AI agent, NPC, product persona, brand voice, or system assistant) used to deliver brand-consistent, policy-bound, compliant behavior inside generative AI, multimodal systems, and autonomous agents.

    1. Identity Properties

    Property Expected Type Description
    name Text Primary name of the SyntheticPersona.
    alternateName Text Alternate or alias names.
    description Text High-level description of what this persona is and does.
    identifier Text | URL | PropertyValue Unique identifier for internal or external reference.
    image ImageObject | URL Avatar or visual identity.
    url URL Canonical documentation URL.
    disambiguatingDescription Text Distinguishes this persona from others with similar names.

    2. Representation & Anchor

    A SyntheticPersona is always an Intangible, but it may represent or simulate entities such as brands, products, systems, or characters. These properties define what the persona acts as and who it acts for.

    Property Expected Type Description
    representsEntity schema:Thing The concrete entity this persona represents (Brand, Product, Organization, Person, DigitalIndividual, System).
    representationKind DefinedTerm | Text The representational mode (“brand-voice”, “product-voice”, “person-like”, “system-assistant”, “npc-character”).

    Examples:
    - Brand persona → "brand-voice"
    - Product persona → "product-voice"
    - NPC → "npc-character"
    - System agent → "system-assistant"

    3. Brand, Product & Organizational Relationships

    Property Expected Type Description
    brand schema:Brand | schema:Organization Brand represented by this persona.
    worksFor schema:Organization Organization that operates or governs the persona.
    associatedProduct schema:Product | schema:SoftwareApplication | schema:Service Product or service this persona is tied to.
    usedInPlatform schema:SoftwareApplication | schema:Service Platforms where persona is deployed.

    4. Audience, Role & Context

    Property Expected Type Description
    personaArchetype DefinedTerm | Text High-level persona category (e.g., “technical expert”).
    intendedAudience schema:Audience | Text Target audience (CTO, developer, consumer, legal).
    expertiseLevel DefinedTerm | Text Depth of understanding (“beginner”, “intermediate”, “expert”).
    domainFocus DefinedTerm | Text Domain areas persona may speak about.
    channelScope DefinedTerm | Text Channels where persona may operate.
    contextGraph CreativeWork | URL Link to the governing brando:Context.

    5. Language, Tone & Emotional Governance

    Property Expected Type Description
    knowsLanguage schema:Language | Text Languages persona may use.
    defaultLanguage schema:Language | Text Primary operating language.
    pronouns DefinedTerm | StructuredValue | Text Pronouns used by persona.
    toneProfile DefinedTerm | Text Governing tone (“technical-neutral”).
    emotionalExpressionPolicy Text | CreativeWork Rules for emotional expression (usually none).
    allowedAffectLevel Text Allowed emotional intensity.
    prohibitedEmotionalAffect DefinedTerm | Text Forbidden affect (e.g., “warm”, “excited”).
    fallbackAffectStrategy Text Emotional fallback mode.
    emotionalVocabularyRules Text | CreativeWork Allowed/banned emotional vocabulary.
    emotionalSafetyBoundaries Text Rules for safe handling of user emotions.

    6. Skills, Tools & Competencies

    6.1 Skills & Abilities

    Property Expected Type Description
    personaSkill DefinedTerm | Text High-level capabilities.
    personaAbility DefinedTerm | Text Action-level abilities.
    domainExpertise DefinedTerm | Text Knowledge domains.
    technicalKnowledge DefinedTerm | Text Technical frameworks known to persona.

    6.2 Tools

    Property Expected Type Description
    authorizedTool schema:SoftwareApplication | URL | Text Tools persona may use.
    prohibitedTool schema:SoftwareApplication | Text Tools persona must not access.
    toolPermissionLevel Text Level of access (read-only, execute-only).
    integrationScope Text APIs/models persona may interface with.

    6.3 Competencies

    Property Expected Type Description
    competencyLevel DefinedTerm | Text Skill proficiency level.
    competencyFramework URL Link to external competency taxonomy.
    hasCredential EducationalOccupationalCredential Credential persona “has”.
    safetyCompetency DefinedTerm Safety competencies (identity-safety, claim-safety).

    7. Verbal Control & Lexical Governance

    7.1 Canonical Vocabulary

    Property Expected Type Description
    canonicalVocabulary DefinedTerm | Text | URL Required vocabulary set.
    lexicalPolicy Text | CreativeWork | URL Governing lexical rules.
    usesVerbalToken brando:VerbalToken Canonical verbal tokens.

    7.2 Prohibited Language

    Property Expected Type Description
    prohibitedVocabulary Text | DefinedTerm Banned words & phrases.
    prohibitedMarketingLanguage Text | DefinedTerm Brand-unsafe hype language.
    prohibitedPattern Text Prohibited syntactic patterns (“no superlatives”).
    profanityPolicy Text | DefinedTerm Profanity rules.

    7.3 Synonym & Term Normalisation

    Property Expected Type Description
    synonymNormalizationStrategy Text | CreativeWork Rules for mapping synonyms to canonical terms.
    forbiddenSynonymBehavior Text Rules prohibiting invented synonyms.
    termSubstitutionRules Text | CreativeWork Explicit substitutions (“say X, never Y”).

    7.4 Claim & Verb Controls

    Property Expected Type Description
    claimVerbConstraints Text | DefinedTerm Allowed/disallowed claim verbs.
    quantifierConstraints Text Restrictions on absolutes.
    regulatoryPhraseRestrictions Text | CreativeWork Regulated phrase limitations.

    7.5 Lexical Policy Linking

    Property Expected Type Description
    lexicalPolicyRef brando:Policy Link to the lexical policy graph node.

    8. Behavioral & Policy Governance

    Property Expected Type Description
    governingPolicyGraph CreativeWork | URL Link to the Brand Policy Graph.
    behaviorPolicy CreativeWork | Text Rules for behavioral output.
    allowedAction DefinedTerm | Action | Text Permitted actions.
    disallowedAction DefinedTerm | Action | Text Forbidden actions.
    complianceScope DefinedTerm | Text Relevant compliance zones.
    refusalStrategy Text | CreativeWork How persona refuses unsafe requests.
    fallbackBehavior Text Behavior when uncertain.
    integrityAuditRules Text | CreativeWork Pre-response safety & identity auditing.

    9. Technical Implementation

    Property Expected Type Description
    underlyingModel schema:SoftwareApplication | Text | URL AI/LLM powering the persona.
    deploymentEnvironment DefinedTerm | Text Environment: prod, staging, dev.
    version Text Persona version string.
    activationCondition Text | CreativeWork When persona activates.
    supportsModality DefinedTerm | Text Modalities supported (text, image, audio).

    10. Lifecycle & Governance Oversight

    Property Expected Type Description
    owner schema:Organization | schema:Person Business owner.
    maintainer schema:Person | schema:Organization Maintains persona configuration.
    dateCreated Date Creation date.
    dateModified Date Last update.
    reviewCycle Duration | Text Required review interval.
    isTestPersona Boolean Whether persona is experimental.

    11. Multi-Agent & Persona Network Relationships

    Property Expected Type Description
    relatedPersona brando:SyntheticPersona Related persona node.
    fallbackPersona brando:SyntheticPersona Persona used if this one cannot respond.
    replacesPersona brando:SyntheticPersona Persona this one replaces.
    isReplacementFor brando:SyntheticPersona Inverse relation.
    worksAlongside brando:SyntheticPersona Multi-agent companions.
