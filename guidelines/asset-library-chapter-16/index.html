    CHAPTER 16 — ASSET LIBRARY & DOWNLOAD RULES

    Brando Brand Guidelines v1.0

    Brando’s Asset Library is the single source of truth for all brand materials: logos, symbols, visual tokens, icons, components, templates, diagrams, and semantic graph assets.

    This chapter defines:

    • What assets exist
    • How they must be organized
    • How they must be downloaded, versioned, and distributed
    • Who may use which assets
    • What modifications are permitted or prohibited

    Brando assets are part of the governance system, not a collection of files.

    16.1 Asset Library Philosophy

    Brando’s asset library is governed by three principles:

    1. Consistency

    Every asset must reflect the Brando identity system without variation.

    2. Precision

    Files must be crisp, structured, versioned, and technically correct.

    3. Control

    Access and usage rules protect the brand’s integrity across platforms and partners.

    16.2 Asset Library Structure

    The official Brando Asset Library is organized into seven core groups:

    1. Logos

    • Primary lockups
    • Stacked lockups
    • Symbol-only marks
    • Gradient hero versions
    • Light/Dark variants

    2. Visual Tokens

    • Semantic node token
    • Edge token
    • Policy bracket icon
    • Aurora ribbon assets

    3. Icon System

    • UI icons
    • Navigation icons
    • Semantic graph icons
    • File/data icons
    • Status icons (success, info, warning, error)

    4. Component System Assets

    • Buttons
    • Tabs
    • Cards
    • Data tables
    • Panels
    • Code block templates

    5. Diagram Templates

    • Brand Policy Graph templates
    • Context mapping templates
    • Category/GTIN mapping templates
    • Policy override diagrams

    6. Imagery & Backgrounds

    • Aurora gradient files
    • Architecture images
    • Minimal technical textures

    7. Templates

    • Web sections
    • Presentation slides
    • Documentation layouts
    • Social graphics
    • Whitepaper covers

    16.3 File Formats & Resolutions

    To ensure universal compatibility:

    Logos

    • SVG (primary)
    • PDF (print)
    • PNG (fallback)
    • ICO (favicon)

    Icons

    • SVG ONLY (Retains crisp geometry across UIs.)

    Components & Templates

    • Figma (.fig)
    • SVG exports
    • PNG previews

    Imagery

    • JPEG (optimized)
    • PNG for transparency
    • WebP for web assets
    • 4K versions for hero visuals

    Diagrams

    • SVG
    • Figma
    • JSON (for semantic graph generation)

    16.4 Naming Conventions

    Files must follow strict naming rules for machine and human readability.

    Logos

    brando-logo-primary.svg

brando-logo-symbol.svg
brando-logo-gradient.svg

    Visual Tokens

    brando-token-node.svg

    Icons

    icon-policy.svg
icon-context.svg

    Templates

    template-hero-01.fig

    Imagery

    bg-aurora-01.jpg

    Consistency ensures predictable retrieval by AI systems.

    16.5 Version Control Rules

    Versioning Format

    v1.0.0 (Major.Minor.Patch)

    Major Versions

    • Identity changes
    • New logo
    • New type system
    • Major structural updates

    Minor Versions

    • New icons
    • New templates
    • Small visual updates

    Patch Versions

    • Fixes to existing files
    • Alignment or contrast adjustments
    • Corrected naming

    Rules

    • Never overwrite previous versions
    • Archive all legacy assets
    • Only the Core Brand Team may publish major or minor versions

    16.6 Asset Access Control

    Tier 1: Full Access (Editable)

    • Core Brand Team
    • Creative Director
    • Design Lead

    Tier 2: Restricted Access (Download Only)

    • Product teams
    • Agencies
    • Partners

    Tier 3: Limited Access (Rendered Assets Only)

    • External developers
    • Clients
    • AI-model implementers

    Public Access

    • Only symbol SVG + primary lockup PNG
    • No editable files
    • No gradients
    • No templates

    16.7 Usage Rules & Permissions

    Allowed

    • Scaling logos proportionally
    • Recoloring between approved light/dark modes
    • Using tokens and icons for diagrams
    • Using templates as provided
    • Exporting for social, web, and slides

    Not Allowed

    • Changing logo proportions
    • Recoloring outside approved palette
    • Adding shadows, bevels, outlines
    • Using aurora gradient logo in UI
    • Mixing icon styles
    • Deleting or modifying semantic nodes in diagrams
    • Combining imagery families incorrectly

    Absolutely Forbidden

    • Creating unauthorized versions of the Brando mark
    • Using Brando assets to represent other brands
    • AI-generated variations of Brando’s core logo
    • Regenerating aurora gradients with inconsistent hues

    16.8 Asset Download Structure

    Assets must be downloadable in a structured hierarchy:

    /brando-assets/
   /logos/
   /visual-tokens/
   /icons/
   /components/
   /templates/
   /imagery/
   /diagrams/
   /json-graphs/

    Each folder includes:

    • README.md (usage rules)
    • metadata.json (version, size, checksum)
    • Files grouped by resolution / variant

    16.9 Metadata Requirements

    Each asset must include metadata for machine-actionable retrieval.

    Metadata Fields

    {
  "name": "brando-logo-primary.svg",
  "version": "1.0.0",
  "type": "logo",
  "category": "primary",
  "createdBy": "Brando Core Design Team",
  "lastUpdated": "2025-01-01",
  "checksum": "<hash>",
  "license": "Brando Internal Use Only"
}

    This enables:

    • AI asset retrieval
    • Version validation
    • Dependency mapping

    16.10 Semantic Asset Linkage (Optional)

    Each asset can optionally be linked to BrandoSchema in JSON-LD:

    {
  "@context": "https://brandoschema.com/",
  "@type": "BrandAsset",
  "name": "Brando Primary Logo",
  "version": "1.0.0",
  "appliesTo": "brando:Brand",
  "assetType": "logo",
  "media": "urn:placeholder:brando/logo-primary"
}

    This prepares Brando for:

    • asset governance in LLMs
    • brand-complete experience across systems
    • semantic enforcement

    16.11 Asset Distribution Rules

    Internal Distribution

    • Use Brando’s official asset registry
    • Never email editable files
    • Use versioned links, not attachments

    External Distribution

    • Approved partners only
    • Light/dark primary lockup only
    • No editable files unless under NDA

    AI Distribution

    • LLMs and agents may use:

    • symbol SVG

    • primary lockup PNG
    • diagram SVGs
    • They may not modify assets
    • They may not generate variations

    16.12 Template Usage Rules

    Templates must be used without alteration to:

    • margins
    • grids
    • typography
    • spacing
    • color palette
    • motion system

    You may change:

    • text
    • images (within allowed families)
    • diagram contents

    You may not change:

    • layout geometry
    • component styling
    • spacing tokens

    16.13 Export Rules

    When exporting assets:

    Web

    • SVG preferred
    • PNG fallback at 2x and 3x
    • WebP for large aurora backgrounds

    Print

    • PDF/X-4
    • CMYK palette
    • No aurora gradients (unless controlled)

    Slides

    • PNG or SVG
    • No compression artifacts

    LLMs

    • Plain SVG or PNG links
    • File names must be intact and canonical

    16.14 Asset Library Integrity Audit

    Before release, asset packs must pass:

    1. Visual Integrity Audit

    • alignment
    • contrast
    • geometry
    • spacing

    2. Semantic Integrity Audit

    • naming
    • vocabulary
    • token usage

    3. Technical Integrity Audit

    • file corruption
    • artifact check
    • correct metadata
    • versioning

    4. Governance Audit

    • adherence to identity
    • no unauthorized variations
    • strict compliance with rules

    16.15 Summary

    Brando’s Asset Library is not simply a set of files. It is the governed repository of Brando’s identity system, built for both humans and AI models.

    The rules ensure:

    • consistency
    • precision
    • controlled access
    • semantic clarity
    • brand safety
    • long-term stability

    Brando’s visual system is a standard — and standards must be protected.
